# M0：12 檔主動式 ETF × 官方揭露對照（盤點表）

> 用途：決定每檔 ETF 的 **發行機構**、**建議揭露入口**與 **自動化 driver**。  
> 機器可讀設定見 [`scripts/config/etf-official-sources.json`](../scripts/config/etf-official-sources.json)。

## source 欄位規範（寫入 `holdings_snapshot.source`）

- **MoneyDJ**：完整來源頁 URL（與先前行為相容）。
- **官方**：`official:<發行機構簡稱>:<ETF代號>:<資料日YYYY-MM-DD>|<選填來源URL或檔名>`  
  例：`official:統一投信:00981A:2026-05-06|https://example.com/holdings.csv`

### 試點 ETF（優先接上 CSV URL）

依 SPEC 選定 **00981A、00994A、00980A** 為試點：在 JSON 將 `pilot` 設為 `true`，並於取得穩定 **可直接下載的 CSV（UTF-8）** 連結後，填入 `csvUrl`、將 `driver` 改為 `csv_url`。未取得連結前維持 `driver: none`，ingest 將自動使用 MoneyDJ。

### MOPS／證交所

公開資訊觀測站（MOPS）與證交所資料適合做 **合規備援或交叉驗證**；完整自動解析成本高，此版不作為預設 driver。
